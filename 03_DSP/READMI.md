# Лабораторная работа 3. DSP-ячейки

## Цель работы

Изучить структуру и функциональные возможности DSP-ячеек в FPGA Xilinx 7, а также научиться использовать их при описании арифметической логики на SystemVerilog.

В рамках лабораторной работы требовалось:
- воспроизвести примеры использования DSP-ячейки;
- провести синтез и сравнить использование ресурсов;
- проанализировать схемы `Synthesis > Schematic`;
- изучить влияние разрядности, типа сброса и режима SIMD;
- разобраться с предварительным сумматором, MAC-режимом и обнаружителем паттернов.

## Краткая теория

DSP-ячейки FPGA Xilinx 7 предназначены для эффективной реализации арифметических операций.  
Они включают в себя:
- умножитель `25x18`;
- предварительный сумматор (`pre-adder`);
- АЛУ с поддержкой режима SIMD;
- обнаружитель паттернов (`pattern detector`);
- аккумулятор и встроенные регистры.

Использовать DSP-ячейки можно двумя способами:
- через примитив `DSP48E1`;
- через обычное HDL-описание логики, которое Vivado автоматически отображает на DSP-ресурсы.

В данной лабораторной работе использовался второй подход, при котором логика описывается на SystemVerilog, а выбор аппаратных ресурсов выполняет САПР.

## Выполненные задания

### 1. Умножение без DSP и с DSP

Были воспроизведены примеры умножения без использования DSP и с использованием DSP-ячейки.

При сравнении схем и использования ресурсов было установлено:
- в примере **без DSP** умножение реализуется с помощью **LUT**, а регистры размещаются в составе **Slice**;
- в примере **с DSP** операция умножения отображается на встроенный аппаратный умножитель DSP.

Vivado по умолчанию старается использовать DSP для умножения, если это не запрещено явно.  
Для запрета используется директива:

```systemverilog
(* use_dsp = "no" *)
```

### 1.2 Изменение разрядности входов

При увеличении разрядности входных операндов одной DSP-ячейки становится недостаточно, так как встроенный умножитель поддерживает размерность не более `25x18` бит.

В результате:
- используются несколько DSP-ячеек;
- появляется дополнительная логика на LUT для объединения частичных результатов.

### 1.3 Почему не видны регистры на схеме

В примере умножения с DSP регистры не отображаются отдельно на схеме синтеза, потому что они встроены внутрь DSP-блока.

### 1.4 Что происходит при асинхронном сбросе

При замене синхронного сброса на асинхронный регистры перестают размещаться внутри DSP-ячейки и выносятся наружу в Slice-регистры.

Это связано с тем, что DSP-блок поддерживает только синхронный сброс встроенных регистров.

### 1.5 Сравнение тактовой частоты

При сравнении вариантов с DSP и без DSP было установлено, что вариант на LUT работает медленнее, так как формирует более длинные комбинационные цепи.

DSP-вариант показывает лучшую тактовую частоту, поскольку:
- использует специализированный аппаратный умножитель;
- содержит встроенные регистры для конвейеризации;
- имеет более оптимизированную внутреннюю структуру для арифметических операций.
